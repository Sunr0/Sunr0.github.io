任务段，任务门练习

发表于 2021-02-08 更新于 2021-02-26 阅读次数： Valine：
本文字数： 5.1k

0x00 任务段

TSS

是一块大小为0X68的内存，用来替换一堆寄存器

TR寄存器

段选择子 属性 Base Limit

这里的Base是指向TSS的地址，Limit则是0x68

任务段描述符

其中需要注意的是此时的G位为0以字节为单位，Type则是未访问的TSS段，综合下来就是XX00E9XX`XXXX0068

在R0可以用LTR为TR寄存器赋值，赋值后Type变成已被使用access，STR则只取其前16位，也就是段选择子

在R3可以用call和jmp去访问任务段

call和jmp在返回的时候细节也不一样

call在返回的时候要看eflag的NT位，NT位为1的时候iret是以Previous Task Link CPU来返回，而NT位为0的时候则是按照当前堆栈pop返回，而断点则会使NT位清0

这里的eflag在断下来之前是0x4002在断下来后是0x2，也就是在断下来后，NT位被清0

而用jmp则eflag的NT位总为0，所以得使用堆栈来返回，我们就得自己保存好调用之前的TR寄存器的值，自己跳转回来，而LTR只是改变寄存器的值并不切换，所以我们又得使用jmp

运行的结果是断下来之前是0x2在断下来后是0x2，所以就验证了jmp得自己跳转回来

0x01 任务门

首先是根据任务门描述符的规则来自己创建一个任务门描述符，来进到1环

0000E500`00490000，写入IDT表中

eq 8003f500 0000E500`00490000

这里注意任务门的DPL为3使在环下有访问的权限

由于GDT表中没有权限在r1的代码段的描述符所以再自己构造一个

CS1:00CFBB00`0000FFFF

eq 8003f0b0 00CFBB00`0000FFFF //16 – B1

SS1:00CFB300`0000FFFF

eq 8003f0b8 00CFB300`0000FFFF //17 – B9

FS1:FFC0B3DF`F0000001

eq 8003f0c0 FFC0B3DF`F0000001 //18 – C1

TSS描述符为XX00A9XX`XXXX0068

eq 8003f0c0 FFC0B3DF`F0000001 //9 – 49

结果是cs：B1 — 10110 0 01，CPL为1，成功进1环
